#### **Distributed Fractal Explorer**
```
🏗️ Architecture: Distributed Computing | Real-time Collaboration
📊 Technologies: Next.js, Firebase, Web Workers, Firestore
```
A groundbreaking project where every website visitor becomes a compute node. Uses Firestore as a distributed job queue where users claim "tiles" of a massive fractal to render. Processed in Web Workers for non-blocking performance, then submitted back to create a collaborative masterpiece.

**Key Features:**
- Serverless function tile assignment
- Web Worker parallel processing
- Real-time progress tracking
- Composite image generation from tiles
- SETI@home inspired architecture

## 🚀 Getting Started

### **Prerequisites**
- Node.js 18+ and npm
- Firebase account (for backend features)
- API keys (optional, for specific features)

### **Installation**

```bash
# Clone the repository
git clone https://github.com/DeepTerrorGG/deepterrorgg-portfolio-website.git
cd deepterrorgg-portfolio-website-1

# Install dependencies
npm install

# Copy environment variables
cp .env.example .env.local

# Configure your API keys in .env.local
# - Firebase credentials
# - Genkit/Google AI API key
# - Resend API key (for contact form)
# - Optional: Spotify, OpenWeatherMap, etc.

# Run development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the portfolio.

### **Environment Variables**

Create a `.env.local` file with:

```env
# Firebase Configuration
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_auth_domain
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_storage_bucket
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id

# AI/Genkit (for AI projects)
GOOGLE_GENAI_API_KEY=your_gemini_api_key

# Email (for contact form)
RESEND_API_KEY=your_resend_api_key

# Optional API keys
SPOTIFY_CLIENT_ID=your_spotify_id
SPOTIFY_CLIENT_SECRET=your_spotify_secret
OPENWEATHER_API_KEY=your_weather_key
```

## 📦 Build & Deploy

### **Build for Production**

```bash
npm run build
npm start
```

### **Deploy to Firebase Hosting**

```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase (if not already done)
firebase init hosting

# Build and deploy
npm run build
firebase deploy
```

### **Deploy to Vercel** (Recommended)

```bash
# Install Vercel CLI
npm install -g vercel

# Deploy
vercel
```

Or connect your GitHub repository to Vercel for automatic deployments.

## 🎯 Key Features Explained

### **Distributed Computing**
The **Distributed Fractal Explorer** uses Firestore as a job queue where visitors become compute nodes, claiming tiles to render and submitting results back to create a collaborative fractal.

### **Real-Time Collaboration**
The **Collaborative Whiteboard** syncs drawing data across clients using Firestore real-time listeners, allowing multiple users to draw simultaneously.

### **AI Integration**
AI projects use **Genkit** with Google's **Gemini** model for text generation, image creation, and interactive experiences. Payment gates protect high-cost AI features.

### **Performance Optimization**
- **React.lazy()** for code splitting
- **Virtualized rendering** in Infinite Canvas
- **useMemo/useCallback** for expensive computations
- **Image optimization** with Next.js Image component
- **Progressive enhancement** for 3D models (desktop only)

### **Accessibility**
- Semantic HTML structure
- ARIA labels on interactive elements
- Keyboard navigation support
- Screen reader compatible
- Proper color contrast ratios
